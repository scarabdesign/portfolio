
* Manual start:
    cd portfolio-api
    npm start
    cd portfolio-public
    npm run dev

* To start public page
    systemctl --user start portfolio@1
    systemctl --user start portfolioapi@1
* Get status
    systemctl --user status portfolio@1
    systemctl --user status portfolioapi@1
* View logs
    journalctl --user-unit portfolio@1
    journalctl --user-unit portfolioapi@1

* Reload daemon after changes
    systemctl --user daemon-reload

* To add another service
    vim ~/.config/systemd/user/name@1.service
    systemctl --user enable name@1

* After updating Node, open up to run on port 80
        > sudo setcap cap_net_bind_service=+ep $(which node)
    Update node link
        > sudo ln $(which node) /usr/bin/node

* To rebuild Docker images:
    docker build -t portfolio-api ~/Projects/portfolio/portfolio-api
    docker build -t portfolio-public ~/Projects/portfolio/portfolio-public

* Create k8 pods/services
    kubectl create -f portfolio-api-pod.yaml
    kubectl create -f portfolio-public-pod.yaml

* Other k8 commands
    kubectl get pods,svc -o wide
    kubectl describe pod portfolio-api
    kubectl describe pod portfolio-public
